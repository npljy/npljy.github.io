js中Map和对象{}的区别

2025-10-24 | 原创 | 前端 FrontEnd Map | 948 字 | 4 分钟
0 条评论

在 JavaScript 中，new Map 和普通对象 {} 都是键值对集合，但它们有重要区别：

1. 键的类型

Map - 键可以是任意类型

对象 - 键只能是字符串或 Symbol

2. 顺序保证

Map - 保持插入顺序

对象 - 不保证顺序（现代JS有改进但仍有差异）

3. 大小获取

Map - 有 size 属性

对象 - 需要手动计算

4. 性能差异

5. 方法和特性对比

Map 方法

对象方法

6. 原型链问题

Map - 没有原型链干扰

对象 - 有原型链

7. 序列化

对象 - 可 JSON 序列化

Map - 不能直接序列化

8. 使用场景总结

使用 Map 的情况：

  • 键类型多样（对象、函数等作为键）
  • 需要频繁增删键值对
  • 需要保持插入顺序
  • 大数据量操作

使用对象的情况：

  • 键都是字符串或 Symbol
  • 需要 JSON 序列化
  • 简单的配置、数据存储
  • 方法和属性的集合

选择依据：如果需要复杂键或高性能操作，用 Map；如果是简单数据结构，用对象。
